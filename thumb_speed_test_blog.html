Thumb Speed Test

A Micro:bit reaction game — how fast is your thumb?

By: Adrien B

Difficulty: Beginner / Intermediate
Language: TypeScript — MakeCode
Hardware: BBC Micro:bit
Game Length: 10 seconds

What Is It?

The Thumb Speed Test is a simple but fun reaction game for the BBC Micro:bit. The goal is to tap Button A as many times as possible in 10 seconds. At the end, your score scrolls across the LED display — challenge your friends to beat it!

How to Play

1
Press A + B together to start Hold both buttons at the same time. All 25 LEDs light up — the game has begun!
2
Tap Button A as fast as you can For the next 10 seconds, press Button A over and over. Every tap adds 1 to your score and plays a tiny beep!
3
Wait for the end sound When 10 seconds are up, the LEDs clear and a low booming tone plays. Time's up — stop pressing!
4
Read your score Your score scrolls across the LED display four times so you can read it easily. That's your tap count!

Block 1 — Scoring with Button A

This block runs every time you press Button A. It checks if the game is on, and if so, adds 1 to your score and plays a short beep.

What each part does:

  • on button A pressed — this hat block listens for Button A. Whenever you press A, everything inside this block runs.
  • if Game_on = 1 then … end — a C-shaped container block. The blue spine on the left and the "end" cap at the bottom show exactly where the if block starts and ends. Only the blocks inside this C-shape run when the condition is true.
  • change score by 1 — each valid tap adds 1 to the score variable. After 10 seconds of tapping, this is your total!
  • play tone Middle G for 1/16 beat in background — plays a quick high-pitched beep so you know your tap was registered, without slowing the game down.

Block 2 — Starting the Game (A + B)

This is the main game block. Pressing A and B together triggers it. It resets everything, runs the 10-second window, then shows your score.

What each part does:

  • on button A+B pressed — only fires when both A and B are pressed together, so you can't accidentally start the game.
  • set score to 0 — resets the score to zero so every new game starts fresh.
  • show leds (all on) — lights up all 25 LEDs as a visual "game on" signal.
  • set timer to 10 — stores the game length (10 seconds) in a variable.
  • set Game_on to 1 — flips the switch to 1, telling Block 1 it's okay to count presses.
  • pause (ms) timer × 1000 — waits 10,000 ms (10 seconds). This is your scoring window!
  • set Game_on to 0 — flips the switch back off. Button A taps no longer count.
  • clear screen — turns off all LEDs to signal the game is over.
  • play tone Low C for 4 beat until done — plays a deep buzzer sound all the way through.
  • repeat 4 times → show number score — scrolls your final score four times using a loop.

Key Coding Ideas Used

  • Event blocks (hat blocks) — onButtonPressed() lets the Micro:bit react instantly to button presses without stopping anything else.
  • Variables as switches — Game_on is used as a flag: 0 = off, 1 = on. A classic game programming pattern.
  • Timing with pause — pause() takes milliseconds, so 10 seconds = 10,000 ms. Multiplying timer × 1000 converts cleanly.
  • Loops — the repeat 4 times loop means we don't have to write show number four separate times.
  • Background sound — playing the beep in background mode means tapping sounds don't interrupt or slow the game.

The Code in JavaScript

Copy and paste the code below into MakeCode's JavaScript tab. It will automatically convert to blocks!

thumb_speed_test.js 
let Game_on = 0
let score = 0
let timer = 0

input.onButtonPressed(Button.A, function () {
    if (Game_on == 1) {
        score += 1
        music.play(
            music.tonePlayable(392, music.beat(BeatFraction.Sixteenth)),
            music.PlaybackMode.InBackground
        )
    }
})

input.onButtonPressed(Button.AB, function () {
    score = 0
    basic.showLeds(`
        # # # # #
        # # # # #
        # # # # #
        # # # # #
        # # # # #
        `)
    timer = 10
    Game_on = 1
    basic.pause(timer * 1000)
    Game_on = 0
    basic.clearScreen()
    music.play(
        music.tonePlayable(131, music.beat(BeatFraction.Breve)),
        music.PlaybackMode.UntilDone
    )
    for (let index = 0; index < 4; index++) {
        basic.showNumber(score)
    }
})

Ideas to Improve It

Add a 3… 2… 1… GO! countdown before the game starts.
Show a smiley face if you beat your last score.
Let the player choose the game length (5, 10, or 20 seconds) using A/B before starting.
Use two Micro:bits connected by radio to compete against a friend.
Use LED brightness to show a progress bar filling up as time runs out.
